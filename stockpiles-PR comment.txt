Notes:
In my latest playthroughs I have observed issues where imported stockpile settings - with for example 64 max bins set on a 3x3 stockpile - would prevent the dwarves from ever starting to haul stuff to the stockpile. Only stockpiles import is able to set more containers than there are available tiles a given  stockpile. The game prevents it. This is why I thought of this feature.
The real fix though could be added after the parsing of the .dfstock file, which could be improved by avoiding setting more containers than there are available tiles in the selected stockpile. It seems a pretty easy fix computing the value with the help of [stockpile].room.extents, but this is C++ and I have 0 clue.
